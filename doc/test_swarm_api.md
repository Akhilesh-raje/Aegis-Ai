# test_swarm_api.py Documentation

## Purpose
`test_swarm_api.py` provides a manual test bench for the "Swarm" multi-persona analysis output. It simulates the raw text response that would be generated by a tiered AI model (NetSec -> Endpoint -> Commander) and verifies that the Decision Engine can correctly extract evidence and actions from this complex narrative.

## Key Features
- **Multi-Persona Narrative Parsing**: Validates the parsing of structured section headers like `### 🌐 [NETSEC_ANALYSIS]`, `### 💻 [ENDPOINT_ANALYSIS]`, and `### 👑 [COMMANDER_DECISION]`.
- **Anomaly Extraction**: Tests the engine's ability to pull out a numbered list of anomalies from the AI text.
- **Confidence Triggering**: Verifies that high `SWARM_CONFIDENCE` (e.g., 0.99) triggers the auto-mitigation logic.

## Implementation Details
- Contains a large multi-line string `response` that serves as the "mock" LLM output.
- Pass this response to `decision_engine.decide_action()` along with a critical event payload.
- Outputs the finalized action, confidence, and captured evidence to the console.

## Dependencies
- `backend.ai.decision_engine`: Used to process the simulated swarm output.

## Usage
```bash
python test_swarm_api.py
```
Use this to debug the Decision Engine's ability to interpret specific LLM response formats.

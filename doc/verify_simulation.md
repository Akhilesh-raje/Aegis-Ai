# verify_simulation.py Documentation

## Purpose
`verify_simulation.py` validates the logic of the `AdversaryEmulationEngine`. It ensures that the engine can correctly start multi-step attack chains and inject the resulting events into the system's event buffer.

## Key Features
- **Attack Chain Execution**: Specifically tests the `ransomware_chain` (MITRE T1486) and `data_exfil_lateral` (MITRE T1041).
- **State Tracking**: Verifies that the engine tracks the "current step" of active simulations.
- **Buffer Injection**: Confirms that events generated by the simulation are correctly appended to the application's global event state.

## Implementation Details
- Initializes `AdversaryEmulationEngine` with a mock `app_state`.
- Uses `asyncio.sleep` to allow the background simulation tasks to progress before inspecting the state.
- Checks both the simulation ID and the number of events injected into the buffer.

## Dependencies
- `backend.engine.adversary_emulation`: The core simulation engine.

## Usage
```bash
python verify_simulation.py
```
Use this to confirm that the training and testing environments (simulations) are functioning as expected.
